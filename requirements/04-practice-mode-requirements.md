# 练习模式扩展需求文档

## 1. 背景

当前产品已经具备“词义选择”玩法：用户按当前级别进入一局 10 题的四选一练习，并获得分数、连击、生命和错词回顾。

本次扩展的目标不是替换现有玩法，而是在现有产品中新增一个并列的“听写”玩法，并通过页面菜单让用户在两种玩法之间切换。

## 2. 目标

本次需求要解决以下问题：

1. 让用户可以在同一套词库下切换不同练习方式。
2. 保持“设置中的水平级别”为全局配置，对所有玩法生效。
3. 在不破坏现有“词义选择”规则的前提下，新增一个独立规则的“听写”模式。

## 3. 范围

### 3.1 本次范围内

- 页面新增练习模式菜单：`词义选择 / 听写`
- 两种模式共用当前所选水平级别
- 听写模式从当前级别词库中抽取单词
- 听写模式提供独立题面、播放流程、输入判题和结果展示
- 浏览器端本地运行，不依赖登录和后端服务

### 3.2 本次范围外

- 不改造现有词义选择模式的回合规则
- 不接入云端账号、排行榜、服务端存档
- 不实现语音识别输入
- 不要求接入第三方在线发音 API
- 不新增句子听写、短语听写、例句翻译等扩展玩法

## 4. 核心需求

### 4.1 全局练习模式菜单

系统需要在页面中提供清晰的练习模式入口，至少包含：

- 词义选择
- 听写

用户切换模式后，应看到与当前模式匹配的玩法说明、开始按钮文案和回合结果信息。

### 4.2 水平级别全局通用

设置中的水平级别为全局配置：

- 用户选择某个级别后，该级别同时作用于词义选择和听写模式
- 切换玩法时，不应重置当前级别
- 两种玩法都应从当前级别的词库中抽词

### 4.3 词义选择模式保持原规则

现有词义选择模式保持原有规则和体验，不因听写模式加入而改变：

- 题量、倒计时、生命、分数、连击等规则保持独立
- 当前已存在的错词回顾、最高分、最近正确率逻辑应继续可用

### 4.4 听写模式规则

听写模式采用独立规则：

1. 每局固定 20 个词
2. 单词来源于当前水平级别词库
3. 每个词自动播放 3 遍
4. 相邻两遍播放之间固定间隔 5 秒
5. 用户根据听到的发音输入英文拼写
6. 系统对输入结果进行判定，并给出正确 / 错误反馈
7. 本局完成后展示听写结果和错词回顾

### 4.5 听写输入与判题

听写模式至少需要包含：

- 当前进度展示
- 当前词的播放状态展示
- 单词输入框
- 提交操作
- 判题反馈

判题的最低要求：

- 忽略大小写差异
- 忽略首尾空格
- 以词库中的标准单词拼写作为正确答案

### 4.6 结果与记录

两种玩法虽然共用级别，但结果统计不应相互污染。

系统应支持按“级别 + 玩法”分别保存和展示：

- 最近表现
- 错词记录
- 与玩法相关的统计信息

## 5. 用户故事

### 用户故事 1

作为一个已经选择了某个英语级别的用户，我希望直接在“词义选择”和“听写”之间切换，而不用反复重新设置级别。

### 用户故事 2

作为一个想练拼写的用户，我希望在当前级别下进行 20 个词的听写，并且每个词会重复播放三遍，方便我完整听清楚。

### 用户故事 3

作为一个已经适应现有词义选择玩法的用户，我希望新增听写模式后，原来的选择题模式仍按原规则运行，不被打乱。

## 6. 约束与注意事项

1. 听写模式的节奏明显慢于词义选择模式，属于独立玩法，不应强行复用当前 75 秒 / 10 题的回合设计。
2. 按“每词 3 遍、间隔 5 秒”计算，听写单局时长会明显长于选择模式，应在文案和交互上给用户预期。
3. 第一版实现应优先考虑浏览器原生可落地能力。
4. 如果浏览器不支持语音播放，应有明确的降级提示，而不是静默失败。

## 7. 验收标准

当满足以下条件时，本次需求可视为完成：

1. 页面上可以明确切换“词义选择”和“听写”两种模式。
2. 用户设置一次水平级别后，两种模式都使用该级别词库。
3. 词义选择模式原有规则和流程不被破坏。
4. 听写模式每局固定 20 个词。
5. 听写模式每个词都会自动播放 3 遍，且播放间隔为 5 秒。
6. 用户可以输入拼写并收到正确 / 错误反馈。
7. 听写模式结束后可以查看本局结果和错词。
8. 两种玩法的统计信息不会混在一起。

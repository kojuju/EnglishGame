# 生存模式（第一阶段）设计文档

## 1. 设计目标

本次设计遵循以下原则：

1. **玩法层级独立**：生存模式作为新的顶级玩法，与练习模式、闯关模式平级
2. **页面物理分离**：生存模式使用独立的 `survival-mode.html` 页面，避免继续堆进现有练习页
3. **设置全局共享**：水平级别仍由统一设置控制，并通过本地存储在不同页面之间共享
4. **统计结构前置扩展**：即便当前只做词义生存，数据结构也先按 `meaning / dictation` 双桶设计

## 2. 总体信息架构

## 2.1 页面拆分

建议保留三个顶级入口页面：

- `practice-mode.html`：练习模式主页与现有练习流程
- `stage-mode.html`：闯关模式主页、闯关进行页、闯关结算页
- `survival-mode.html`：生存模式主页、生存进行页、生存结果页

### 这样拆分的原因

1. 练习模式已经有自己的“玩法切换 + 回合逻辑 + 结果页”
2. 闯关模式已经有自己的“关卡选择 + 解锁进度 + 结算规则”
3. 生存模式新增后，会引入“无限续题 + 单命终局 + 纪录挑战”这套独立逻辑
4. 如果继续把练习玩法挂在入口页上，页面状态和统计语义会继续膨胀

## 2.2 左侧侧边栏

建议将三个页面的左侧导航统一为：

- 设置
- 练习模式
- 闯关模式
- 生存模式

交互建议：

- 在 `practice-mode.html` 中，`练习模式` 为当前高亮
- 在 `stage-mode.html` 中，`闯关模式` 为当前高亮
- 在 `survival-mode.html` 中，`生存模式` 为当前高亮
- `设置` 在三个页面中都调用同一套设置弹窗逻辑

## 3. 生存模式页面结构

`survival-mode.html` 建议包含三个 screen：

1. **生存首页**
2. **生存进行页**
3. **生存结果页**

### 3.1 生存首页

生存首页建议展示：

- 当前级别
- 当前玩法：词义生存
- 当前级别历史最佳纪录
- 当前级别最近一次表现
- 开始挑战按钮
- 最近错词回顾入口

首页核心任务是让用户快速理解：

- 这是一个独立玩法
- 规则是“不限题量，答错即结束”
- 当前级别下已经打到过多少题

### 3.2 生存进行页

生存进行页复用现有词义题面的核心元素：

- 当前题号：如 `第 12 题`
- 分数
- 连续答对题数
- 当前级别最佳纪录
- 英文单词
- 4 个中文选项
- 答题反馈
- 退出本局

第一阶段建议不展示：

- 整局倒计时
- 多生命值
- 关卡编号

### 3.3 生存结果页

生存结果页建议展示：

- 本局坚持答对题数
- 本局总作答数
- 本局正确率
- 本局最终得分
- 当前级别最佳纪录
- 是否刷新纪录
- 本局错词回顾
- 操作按钮：`再来一局`、`返回生存首页`

## 4. 生存回合设计

## 4.1 基础规则

第一阶段生存模式只做词义选择，建议默认配置：

```js
const SURVIVAL_CONFIG = {
  meaning: {
    startingLives: 1,
    baseScore: 100,
    feedbackDelayMs: 850,
    recentWordWindowSize: 12
  }
};
```

说明：

- `startingLives: 1` 只是实现层表达，产品语义等价于“答错即结束”
- `recentWordWindowSize` 用于避免短时间重复同一单词

## 4.2 单局流程

建议单局流程如下：

1. 用户从首页进入生存模式
2. 系统根据当前级别初始化出题队列
3. 渲染第 1 题
4. 用户作答
5. 若答对，累计分数和连对数，并立即补出下一题
6. 若答错，展示纠错反馈并结束本局
7. 进入结果页并保存本局统计

## 4.3 终局条件

建议终局条件只有以下一种：

1. 用户在当前局中答错任意一题

退出本局则视为主动放弃，不建议写入最佳纪录，但可以选择不更新最近成绩。

## 5. 无限续题策略

## 5.1 题目来源

第一阶段词义生存所有题目都来源于当前级别完整词库。

生成策略建议：

- 正确答案来自当前级别词库中的真实单词
- 干扰项从当前级别完整词库中抽取
- 题面结构继续复用 `createMeaningQuestion`

## 5.2 队列机制

建议为当前局维护一个“待出题队列”：

1. 开局时，用当前级别词库生成一轮洗牌队列
2. 每答对一题，从队列头部取下一题
3. 队列耗尽后，用同一词库重新洗牌并续上新队列
4. 每次开始新的一局生存时，都必须重新生成新的洗牌队列，不能复用上一局的题目顺序

这样可以做到：

- 单局真正不限题量
- 不需要一次性提前构建整局题目数组
- 易于控制短期去重策略

## 5.3 短期去重

为避免用户在短时间内遇到同一单词，建议额外维护一个最近出题窗口。

例如：

- 保存最近 12 个已出单词 ID
- 新补题时，尽量避开这些单词
- 如果词库规模过小，再允许回退到普通洗牌策略

## 5.4 跨局混淆与选项重抽

为了保证生存模式的挑战性，两把独立生存局之间不应形成固定题包记忆。

建议约束如下：

1. 新开一局时，必须重新洗牌当前级别词库，避免题目顺序与上一局一致
2. 每次生成题面时，都重新调用 `createMeaningQuestion` 生成选项
3. 同一单词在不同局里，正确答案位置不应固定
4. 同一单词在不同局里，干扰项组合应尽量变化，避免用户只记住上一局的四个选项

这套约束与闯关模式的“固定关卡词包”不同。  
生存模式只固定“题目来源级别”，不固定“跨局顺序和选项排列”。

## 6. 状态与本地存储设计

## 6.1 建议新增状态字段

```js
state.selectedLevel = "cet4";
state.selectedMode = "meaning";
state.questionQueue = [];
state.recentWordIds = [];
state.currentQuestion = null;
state.answeredCount = 0;
state.correctCount = 0;
state.score = 0;
state.bestRecord = 0;
state.wrongWords = [];
state.roundFinished = false;
```

说明：

- `selectedMode` 先固定为 `meaning`，但保留字段为未来听写生存预留
- `questionQueue` 用于承载无限续题
- `recentWordIds` 用于短期去重

## 6.2 本地存储设计

不要把生存模式混进当前练习模式的 `statsByModeAndLevel` 或闯关模式的 `stageProgressByModeAndLevel` 里。  
建议在 `shared-game.js` 中新增一组独立键：

- `survivalStatsByModeAndLevel`
- `survivalWrongWordsByModeAndLevel`

键结构示例：

```text
englishGame.survivalStatsByModeAndLevel
englishGame.survivalWrongWordsByModeAndLevel
```

数据组织示例：

```json
{
  "meaning": {
    "cet4": {
      "bestStreak": 27,
      "bestScore": 3120,
      "latestAccuracy": 96,
      "latestAnsweredCount": 28
    }
  },
  "dictation": {}
}
```

## 6.3 错词记录

第一阶段建议继续沿用现有错词回顾结构：

- 记录单词 ID
- 记录单词本体、词义、词性、音标
- 记录作答时的原始选项

如果本局只有最后一题答错，那么错词列表通常只会包含该终结词。  
后续如果想扩展为“显示本局所有曾答错的词”，也不需要推翻当前结构。

## 7. 结果与纪录设计

## 7.1 主指标选择

第一阶段建议把以下指标作为主展示：

1. **坚持答对题数**
2. **历史最佳纪录**

原因：

- 生存玩法的核心吸引力在于“这次坚持了多少题”
- 这比沿用练习模式的“最高分”更符合用户直觉

## 7.2 分数的角色

分数仍可保留，但定位为辅助指标：

- 答对继续加分
- 可保留连对奖励
- 不应喧宾夺主，替代“纪录挑战”本身

## 8. 代码落点

建议涉及以下文件：

- `practice-mode.html`：补左侧“生存模式”导航
- `stage-mode.html`：补左侧“生存模式”导航
- `survival-mode.html`：新增页面结构
- `styles.css`：复用现有样式，必要时补少量生存页专属样式
- `shared-game.js`：补生存模式配置、存储 key 和读写 helper
- `survival-mode.js`：实现首页、游戏流程、结果页和本地持久化

## 9. 实施边界

### 9.1 与练习模式的边界

- 不把生存模式作为练习模式页内的新标签页
- 不复用练习模式“固定 10 题一局”的回合假设
- 不把练习模式成绩自动换算为生存纪录

### 9.2 与闯关模式的边界

- 不引入关卡、解锁、通关、失败原因这类闯关语义
- 不写入闯关进度结构
- 不让生存模式结果影响闯关解锁

### 9.3 首期收敛范围

第一阶段建议刻意收窄为：

- 顶级入口成立
- 词义生存可玩
- 无限续题成立
- 纪录统计成立

暂不追求：

- 听写生存
- 排行榜
- 每日挑战
- 战绩列表
- 分享链路

# 练习模式扩展设计文档

## 1. 设计目标

本次设计遵循两个原则：

1. **玩法并列，不互相侵入**：词义选择与听写共享词库级别，但各自拥有独立回合逻辑。
2. **优先复用现有页面骨架**：继续使用首页 / 游戏页 / 结果页三段式结构，在同一页面框架下按模式切换内容。

## 2. 信息架构

### 2.1 全局状态

新增一个全局练习模式状态：

- `meaning`：词义选择
- `dictation`：听写

现有全局级别状态继续保留：

- `selectedLevel`

两者关系：

- `selectedLevel` 决定两种玩法共同使用的词库范围
- `selectedMode` 决定首页、游戏页、结果页当前展示哪套玩法 UI

### 2.2 页面结构

仍然保留三块主页面：

- 首页
- 游戏页
- 结果页

在页面顶部或首页主操作区新增“练习模式菜单”，作为一级切换入口。

## 3. 交互设计

## 3.1 首页

首页新增内容：

- 练习模式菜单
- 与当前模式匹配的标题、副文案、开始按钮文案

推荐表现形式：

- 两个并列按钮或标签页：`词义选择`、`听写`
- 当前选中模式有明显高亮

首页动态文案建议：

- 词义选择：`10 题一局，按你的水平练词义`
- 听写：`20 个词一局，听发音写拼写`

### 3.2 设置

设置弹窗继续只负责“水平级别”配置，不增加模式配置。

这样做的理由：

- 模式切换属于更高频操作，应放在主界面而不是设置中
- 级别属于全局学习难度，适合作为共享配置

### 3.3 游戏页

游戏页按模式渲染两套不同内容。

#### 词义选择模式

保持现有设计：

- 展示英文单词
- 展示音标和词性
- 展示 4 个中文选项
- 使用当前已有分数、生命、连击、倒计时

#### 听写模式

采用独立题面：

- 展示当前进度：如 `第 3 / 20 词`
- 展示当前播放进度：如 `第 2 / 3 遍`
- 展示播放状态：`播放中 / 等待下一遍 / 等待提交`
- 展示输入框
- 展示提交按钮
- 展示当前级别信息

听写模式不展示当前英文单词本体，也不展示音标。

### 3.4 结果页

结果页按模式展示不同统计：

#### 词义选择模式

继续展示：

- 分数
- 正确题数
- 正确率
- 最高连击
- 错词回顾

#### 听写模式

建议展示：

- 正确词数
- 总词数
- 正确率
- 错词回顾

第一版不强制要求与选择题共享“分数 / 连击 / 生命”概念。

## 4. 状态设计

## 4.1 建议新增状态字段

```js
state.selectedMode = "meaning" | "dictation";
state.dictationInput = "";
state.dictationRound = [];
state.dictationPlaybackCount = 0;
state.dictationPlaybackStatus = "idle" | "speaking" | "waiting" | "ready";
state.dictationCorrectCount = 0;
state.dictationWrongWords = [];
state.dictationSubmittingLocked = false;
state.dictationTimerId = null;
```

说明：

- `selectedMode`：当前玩法
- `dictationRound`：当前听写局的 20 个词
- `dictationPlaybackCount`：当前词已播放次数
- `dictationPlaybackStatus`：当前词播放阶段
- `dictationCorrectCount`：听写答对数
- `dictationWrongWords`：听写错词列表

## 4.2 本地存储设计

建议新增或改造存储结构，使统计按“模式 + 级别”隔离：

- `selectedMode`
- `statsByModeAndLevel`
- `wrongWordsByModeAndLevel`

键结构示例：

```text
englishGame.selectedMode
englishGame.statsByModeAndLevel
englishGame.wrongWordsByModeAndLevel
```

数据组织示例：

```json
{
  "meaning": {
    "cet4": { "bestScore": 1200, "latestAccuracy": 80 }
  },
  "dictation": {
    "cet4": { "latestAccuracy": 65, "correctCount": 13 }
  }
}
```

## 5. 听写模式回合设计

### 5.1 回合规则

听写模式采用独立节奏：

- 每局固定 20 个词
- 不复用词义选择模式的 10 题 / 75 秒 / 3 生命规则
- 一局必须按顺序完成 20 个词后结算

### 5.2 单词播放流程

单个词的流程建议如下：

1. 进入新词
2. 自动播放第 1 遍
3. 等待 5 秒
4. 自动播放第 2 遍
5. 等待 5 秒
6. 自动播放第 3 遍
7. 播放结束后进入可提交状态
8. 用户提交答案
9. 系统反馈正确或错误
10. 进入下一个词

### 5.3 输入策略

第一版建议：

- 输入框从第一遍播放开始就可输入
- 提交按钮在第三遍播放结束后可用
- 用户提交后才进入下一词

这样既保留“边听边写”的自然感，也保证“每词固定播放三遍”的需求被满足。

### 5.4 判题策略

第一版采用严格但可预期的规则：

- 转小写后比较
- 去除首尾空格后比较
- 不做模糊匹配
- 不自动容错近似拼写

示例：

- `Apple` 与 `apple` 视为正确
- ` apple ` 视为正确
- `aple` 视为错误

## 6. 音频方案

## 6.1 第一版方案

第一版优先使用浏览器原生 `speechSynthesis`：

- 无需后端
- 无需额外音频资源
- 实现成本最低

推荐参数：

- `lang`: `en-GB` 优先，找不到时回退到可用英语语音
- `rate`: 略慢于默认值，便于听写

### 6.2 限制

浏览器 TTS 存在以下不确定性：

- 不同浏览器音色不同
- 不同设备英语语音包可用性不同
- 某些环境首次调用可能有延迟

### 6.3 降级策略

如果浏览器不支持语音播放，建议：

1. 在听写模式入口或开始前展示明确提示
2. 阻止进入无声的听写流程
3. 不影响词义选择模式正常使用

## 7. 节奏评估

按照本次规则，听写模式单局节奏明显慢于选择题模式。

粗略估算：

- 每个词至少包含 3 次发音
- 两个固定间隔，共 10 秒
- 再加上用户输入与提交时间

因此 20 个词一局的实际时长通常会在数分钟级别。这是设计上的有意取舍，不应继续沿用“短平快选择题”的节奏预期。

## 8. 错词回顾设计

听写模式的错词回顾建议展示：

- 正确单词
- 音标
- 词性
- 中文释义

如后续实现成本允许，可额外展示：

- 用户错误输入

第一版若不记录用户原始输入，也不影响核心功能上线。

## 9. 与现有代码的对接边界

本次实现建议复用：

- 现有 `selectedLevel`
- 现有词库结构
- 现有首页 / 游戏页 / 结果页切换机制
- 现有错词回顾渲染能力
- 现有本地存储辅助函数

本次实现建议新增：

- 模式菜单 UI
- 模式状态管理
- 听写专用回合构建函数
- 听写专用题面渲染函数
- 听写专用提交判题流程
- 语音播放调度逻辑
- 按模式隔离的统计存储

## 10. 建议实现顺序

1. 增加全局模式状态和菜单 UI
2. 让首页、游戏页、结果页支持按模式切换文案和布局
3. 将现有词义选择逻辑包裹为 `meaning` 模式
4. 新增听写模式的 20 词回合构建逻辑
5. 接入浏览器 TTS 和三遍播放调度
6. 完成输入、判题、反馈、结算
7. 将统计和错词记录改为按“模式 + 级别”存储

## 11. 关键开放项

以下问题当前有建议方向，但实现前仍可再确认：

1. 听写模式是否需要“手动重播一次”按钮
2. 听写模式结果页是否需要分数概念，还是只保留正确率
3. 是否要记录用户的错误拼写以便后续复盘
4. 听写模式首页是否要明确提示“单局时间更长”
